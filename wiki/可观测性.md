# 可观测性

---

## 概述

BaseBackend 的可观测性体系基于 OpenTelemetry + Micrometer，集成 Prometheus（指标）、Loki（日志）、Tempo（链路追踪）、Grafana（可视化）。

核心模块：
- `basebackend-observability`：可观测性基础设施库
- `basebackend-observability-service`：可观测性数据聚合微服务

---

## 三大支柱

```
                      可观测性
                         │
           ┌─────────────┼─────────────┐
           │             │             │
      Metrics        Logging        Tracing
     (指标监控)      (日志聚合)     (链路追踪)
           │             │             │
      Prometheus       Loki          Tempo
           │             │             │
           └─────────────┼─────────────┘
                         │
                      Grafana
                    (统一可视化)
```

## OpenTelemetry 集成

### 配置

```yaml
# application-observability.yml
management:
  endpoints:
    web:
      exposure:
        include: '*'
  endpoint:
    health:
      show-details: always
  metrics:
    tags:
      application: ${spring.application.name}
  tracing:
    sampling:
      probability: 1.0    # 采样率（生产建议 0.1）

otel:
  exporter:
    otlp:
      endpoint: http://localhost:4318
  resource:
    attributes:
      service.name: ${spring.application.name}
      service.version: 1.0.0
      deployment.environment: ${spring.profiles.active:dev}
```

### 自动埋点

OpenTelemetry Java Agent 自动采集：
- HTTP 请求（Spring MVC / WebFlux）
- 数据库查询（JDBC / MyBatis）
- Redis 操作
- RocketMQ 消息
- Feign RPC 调用

## Prometheus 指标

### 内置指标

| 指标类别 | 示例指标 | 说明 |
|---------|---------|------|
| JVM | `jvm_memory_used_bytes` | JVM 内存使用 |
| JVM | `jvm_threads_live_threads` | 活跃线程数 |
| JVM | `jvm_gc_pause_seconds` | GC 暂停时间 |
| HTTP | `http_server_requests_seconds` | HTTP 请求延迟 |
| HTTP | `http_server_requests_seconds_count` | HTTP 请求计数 |
| DB | `hikaricp_connections_active` | 数据库活跃连接数 |
| Cache | `cache_gets_total` | 缓存命中次数 |
| MQ | `rocketmq_producer_send_total` | 消息发送总数 |

### 自定义指标

```java
@Component
public class BusinessMetrics {

    private final Counter orderCounter;
    private final Timer orderProcessTimer;

    public BusinessMetrics(MeterRegistry registry) {
        this.orderCounter = Counter.builder("business.orders.created")
            .description("Number of orders created")
            .tag("service", "order")
            .register(registry);

        this.orderProcessTimer = Timer.builder("business.orders.process.time")
            .description("Order processing time")
            .register(registry);
    }

    public void recordOrderCreated() {
        orderCounter.increment();
    }

    public void recordProcessTime(Runnable task) {
        orderProcessTimer.record(task);
    }
}
```

### 指标端点

```bash
# 访问 Prometheus 指标
curl http://localhost:8081/actuator/prometheus
```

## Loki 日志聚合

### 日志格式

结构化 JSON 日志输出，便于 Loki 索引：

```json
{
  "timestamp": "2024-01-15T10:30:00.000Z",
  "level": "INFO",
  "logger": "com.basebackend.user.service.UserServiceImpl",
  "message": "User created successfully",
  "traceId": "abc123def456",
  "spanId": "789xyz",
  "userId": 1001,
  "service": "basebackend-user-api"
}
```

### 日志查询（LogQL）

```
# 查询用户服务的错误日志
{service="basebackend-user-api"} |= "ERROR"

# 查询特定 TraceId 的所有日志
{service=~"basebackend-.*"} |= "traceId=abc123def456"

# 查询慢请求日志
{service="basebackend-gateway"} | json | duration > 1000
```

## Tempo 链路追踪

### 链路传播

```
前端请求 → Gateway → User API → MySQL
              │         │
              │         └──→ Redis
              │
              └──→ System API → MySQL
```

每个请求生成唯一的 TraceId，贯穿整个调用链。

### TraceId 传播

- Gateway 的 `TraceIdFilter` 在请求头中注入 `X-Trace-Id`
- 下游微服务自动从请求头提取 TraceId
- 日志中自动注入 TraceId，便于关联查询

### 查询链路

通过 Grafana Tempo 数据源查询：
1. 输入 TraceId
2. 查看完整调用链路图
3. 定位耗时最长的 Span

## Grafana 仪表盘

### 推荐仪表盘

| 仪表盘 | 数据源 | 展示内容 |
|--------|--------|---------|
| JVM Overview | Prometheus | JVM 内存、GC、线程 |
| HTTP Overview | Prometheus | 请求量、延迟、错误率 |
| MySQL Overview | Prometheus | 连接池、查询延迟、慢查询 |
| Redis Overview | Prometheus | 命中率、内存、连接数 |
| RocketMQ Overview | Prometheus | 消息发送/消费量、延迟 |
| Service Map | Tempo | 服务依赖拓扑 |
| Log Explorer | Loki | 日志搜索与分析 |

### 访问地址

| 服务 | 地址 | 默认账号 |
|------|------|---------|
| Grafana | http://localhost:3000 | admin / admin |
| Prometheus | http://localhost:9090 | - |
