# 微服务模块说明

---

## 服务总览

| 服务 | 端口 | Spring 应用名 | 职责 |
|------|------|--------------|------|
| Gateway | 8180 | `basebackend-gateway` | API 网关，统一入口 |
| User API | 8081 | `basebackend-user-api` | 用户认证与管理 |
| System API | 8082 | `basebackend-system-api` | 系统管理 |
| Notification Service | - | `basebackend-notification-service` | 通知消息 |
| Observability Service | - | `basebackend-observability-service` | 可观测性 |
| File Service | - | `basebackend-file-service` | 文件存储 |

---

## Gateway 网关服务

### 职责
- 统一入口，所有外部请求通过 Gateway 转发到后端微服务
- 认证鉴权（JWT Token 验证）
- 请求路由（基于 Nacos 服务发现 + 静态路由配置）
- 灰度发布（基于 Header / Weight / IP / User 策略）
- 熔断限流（Resilience4j）
- 安全防护（头部清理、安全头注入、签名验证）

### 过滤器链

过滤器按执行顺序排列：

| 序号 | 过滤器 | 职责 |
|------|--------|------|
| 1 | `AuthenticationFilter` | JWT Token 验证，白名单放行 |
| 2 | `HeaderSanitizationFilter` | 清理危险请求头，防止头注入攻击 |
| 3 | `SecurityHeadersFilter` | 添加安全响应头（CSP、X-Frame-Options 等） |
| 4 | `SignatureVerifyFilter` | API 签名验证（支付/订单等敏感接口） |
| 5 | `RequestSizeLimitFilter` | 请求体大小限制 |
| 6 | `IdempotencyFilter` | 幂等性检查（防重复提交） |
| 7 | `ApiVersionFilter` | API 版本路由 |
| 8 | `SlowRequestFilter` | 慢请求检测与告警 |
| 9 | `AccessLogFilter` | 访问日志记录 |
| 10 | `ResponseCacheFilter` | 响应缓存 |
| 11 | `TraceIdFilter` | 链路追踪 ID 注入 |
| 12 | `SeataGatewayFilterConfig` | Seata 分布式事务上下文传播 |

### 路由配置

当前默认生效的静态路由定义在 `application-gateway.yml` 中，
`application-routes.yml` 仅保留候选路由示例：

| 路由 ID | 路径匹配 | 目标服务 | 熔断器 |
|---------|---------|---------|--------|
| `user-api` | `/basebackend-user-api/**` | `lb://basebackend-user-api` | `userServiceCircuitBreaker` |
| `system-api` | `/basebackend-system-api/**` | `lb://basebackend-system-api` | `systemServiceCircuitBreaker` |
| `notification-service` | `/api/notifications/**` | `lb://basebackend-notification-service` | `notificationServiceCircuitBreaker` |
| `file-service` | `/api/files/**` | `lb://basebackend-file-service` | `fileServiceCircuitBreaker` |
| `observability-*` | `/api/metrics/**`, `/api/traces/**`, `/api/logs/**`, `/api/alerts/**` | `lb://basebackend-observability-service` | `observabilityServiceCircuitBreaker` |

### 灰度发布配置

```yaml
gateway:
  gray:
    enabled: true
    rules: []  # 默认不内置灰度规则，按实际服务动态配置
```

### 熔断器配置

```yaml
resilience4j:
  circuitbreaker:
    configs:
      default:
        slidingWindowSize: 10
        minimumNumberOfCalls: 5
        permittedNumberOfCallsInHalfOpenState: 3
        waitDurationInOpenState: 5s
        failureRateThreshold: 50
  timelimiter:
    configs:
      default:
        timeoutDuration: 10s
```

## User API 用户服务

### 职责
- 用户认证（登录、注册、Token 刷新、登出）
- 用户管理（CRUD、状态管理）
- 角色管理（角色分配、权限关联）
- 个人资料管理
- 用户偏好设置
- 安全管理（密码修改、设备管理）

### Controller 列表

| Controller | 路径前缀 | 职责 |
|-----------|---------|------|
| `AuthController` | `/api/user/auth` | 用户认证（登录/注册/刷新/登出） |
| `UserController` | `/api/user/users` | 用户 CRUD 管理 |
| `RoleController` | `/api/user/roles` | 角色管理 |
| `ProfileController` | `/api/user/profile` | 个人资料 |
| `PreferenceController` | `/api/user/preferences` | 用户偏好 |
| `SecurityController` | `/api/user/security` | 安全设置 |

### 主要 API 端点

#### 认证接口

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/user/auth/login` | 用户登录 | 否 |
| POST | `/api/user/auth/register` | 用户注册 | 否 |
| POST | `/api/user/auth/refresh` | 刷新 Token | 否（需 Refresh Token） |
| POST | `/api/user/auth/logout` | 用户登出 | 是 |

#### 用户管理接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/user/users` | 用户列表（分页） | `user:list` |
| GET | `/api/user/users/{id}` | 用户详情 | `user:query` |
| POST | `/api/user/users` | 创建用户 | `user:add` |
| PUT | `/api/user/users/{id}` | 更新用户 | `user:edit` |
| DELETE | `/api/user/users/{id}` | 删除用户 | `user:delete` |

#### 角色管理接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/user/roles` | 角色列表 | `role:list` |
| POST | `/api/user/roles` | 创建角色 | `role:add` |
| PUT | `/api/user/roles/{id}` | 更新角色 | `role:edit` |
| DELETE | `/api/user/roles/{id}` | 删除角色 | `role:delete` |

### 配置

```yaml
spring:
  application:
    name: basebackend-user-api
server:
  port: 8081
```

## System API 系统管理服务

### 职责
- 部门管理（组织架构树）
- 字典管理（数据字典）
- 操作日志
- 系统监控
- 权限管理
- 应用管理
- 列表通用操作
- 对外 OpenAPI

### Controller 列表

| Controller | 路径前缀 | 职责 |
|-----------|---------|------|
| `DeptController` | `/api/system/depts` | 部门管理 |
| `DictController` | `/api/system/dicts` 或 `/api/dicts` | 字典管理 |
| `LogController` | `/api/system/logs` | 操作日志 |
| `MonitorController` | `/api/system/monitor` | 系统监控 |
| `PermissionController` | `/api/system/permissions` | 权限管理 |
| `ApplicationController` | `/api/system/application` | 应用管理 |
| `ApplicationResourceController` | `/api/system/application/resources` | 应用资源管理 |
| `ListOperationController` | `/api/system/list` | 列表通用操作 |
| `OpenApiController` | `/api/system/open` | 对外开放接口 |

### 配置

```yaml
spring:
  application:
    name: basebackend-system-api
server:
  port: 8082
```

## Notification Service 通知服务

### 职责
- 站内通知
- 邮件通知
- SSE（Server-Sent Events）实时推送
- 通知模板管理

### 路由
- 标准路由：`/api/notifications/**`
- SSE 连接：`/api/notifications/stream`

## Observability Service 可观测性服务

### 职责
- 指标数据聚合与查询
- 链路追踪数据查询
- 日志数据查询
- 告警管理

### 路由
- 指标：`/api/metrics/**`
- 链路：`/api/traces/**`
- 日志：`/api/logs/**`
- 告警：`/api/alerts/**`

## File Service 文件服务

### 职责

- 文件元数据管理

### 路由
- `/api/files/**`
