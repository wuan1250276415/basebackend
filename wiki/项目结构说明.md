### 依赖层级（从底到上）

| 层级 | 模块 | 说明 |
|------|------|------|
| L0 | `common-core`, `common-util` | 零外部依赖的基础工具 |
| L1 | `common-context`, `common-security`, `common-dto` | 依赖 L0 |
| L2 | `common-lock`, `common-idempotent`, `common-datascope`, `common-ratelimit`, `common-export`, `common-event` | 功能型公共模块 |
| L3 | `jwt`, `database`, `cache`, `logging`, `security`, `messaging`, `transaction`, `nacos`, `observability` | 基础设施库 |
| L4 | `security-starter`, `common-starter`, `web`, `feign-api`, `feature-toggle` | Starter / 接口定义 |
| L5 | `gateway`, `user-api`, `system-api`, `notification-service`, `observability-service`, `file-service` | 可部署微服务 |

## 各模块职责说明

### 公共模块（common）

| 子模块 | 职责 |
|--------|------|
| `common-core` | 统一响应（`Result<T>`、`PageResult<T>`）、分页查询（`PageQuery`）、异常体系（`BusinessException`、`ErrorCode`）、基础实体（`BaseEntity`） |
| `common-dto` | 跨服务共享的 DTO 定义 |
| `common-util` | 通用工具类 |
| `common-context` | 用户上下文（`UserContextHolder`）、租户上下文（`TenantContextHolder`） |
| `common-security` | 安全相关工具 |
| `common-starter` | 自动配置聚合 |
| `common-storage` | 文件存储 SPI（Local / MinIO / S3 / OSS 四种 Provider） |
| `common-lock` | `@DistributedLock` 注解 + SpEL + Redis/内存双实现 |
| `common-idempotent` | `@Idempotent` 注解 + TOKEN/PARAM/SPEL 三策略 |
| `common-datascope` | `@DataScope` 注解 + MyBatis 拦截器 |
| `common-ratelimit` | `@RateLimit` 注解 + 滑动窗口/令牌桶/固定窗口三种算法 |
| `common-export` | CSV + EasyExcel 导出导入 + 异步大数据量导出 |
| `common-event` | DomainEvent + EventStore + 可靠发布 + 重试 + 指数退避 |

### 基础设施库

| 模块 | 职责 |
|------|------|
| `jwt` | JWT 令牌生成/验证/刷新，双 Token 机制，密钥轮换，黑名单，多设备管理 |
| `database` | MyBatis Plus 配置，Flyway 迁移，动态数据源，多租户，审计日志，敏感数据加密 |
| `cache` | 多级缓存，分布式锁，布隆过滤器，热 Key 检测，缓存预热 |
| `security` | Spring Security 集成，动态权限，XSS 防护，密钥管理 |
| `messaging` | RocketMQ 集成，事务消息，幂等消费，死信队列 |
| `transaction` | Seata AT 模式封装 |
| `nacos` | 配置管理，服务发现，灰度发布 |
| `observability` | OpenTelemetry 集成，Metrics / Tracing / Logging |
| `feature-toggle` | Unleash / Flagsmith Feature Toggle |

### 可部署微服务

| 服务 | 端口 | 职责 |
|------|------|------|
| `gateway` | 8180 | API 网关，统一入口，认证过滤，路由转发，限流 |
| `user-api` | 8081 | 用户认证，用户管理，角色管理，个人资料，安全设置 |
| `system-api` | 8082 | 部门管理，字典管理，日志管理，监控，权限管理，应用管理 |
| `notification-service` | - | 通知服务，消息模板 |
| `observability-service` | - | 可观测性数据聚合 |
| `file-service` | - | 文件上传/下载，多存储 Provider |

## 标准微服务内部结构

每个微服务遵循统一的分层架构：

```
basebackend-xxx-api/
├── src/main/java/com/basebackend/xxx/
│   ├── controller/          # 控制层（接收请求，参数校验）
│   ├── service/             # 服务层（业务逻辑）
│   │   └── impl/            #   服务实现
│   ├── mapper/              # 数据访问层（MyBatis Mapper 接口）
│   ├── entity/              # 数据库实体
│   ├── dto/                 # 数据传输对象
│   ├── vo/                  # 视图对象
│   ├── config/              # 配置类
│   └── XxxApplication.java  # 启动类
├── src/main/resources/
│   ├── application.yml      # 主配置
│   ├── application-dev.yml  # 开发环境配置
│   ├── mapper/              # MyBatis XML
│   └── db/migration/        # Flyway 迁移脚本
└── pom.xml
```
