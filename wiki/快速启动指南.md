# 快速启动指南

---

## 5 分钟快速启动

### 前置条件

确保以下服务已启动：
- MySQL 8（端口 3306）
- Redis（端口 6379）

### 第一步：克隆项目

```bash
git clone https://github.com/wuan1250276415/basebackend.git
cd basebackend
```

### 第二步：启动基础设施

```bash
# 使用 Docker Compose 启动 MySQL + Redis
cd docker/compose/base
docker-compose up -d
cd ../../..
```

### 第三步：构建项目

```bash
mvn clean install -DskipTests
```

### 第四步：启动微服务

```bash
# 终端 1：启动 Gateway（端口 8180）
cd basebackend-gateway
mvn spring-boot:run -Dspring-boot.run.profiles=dev
cd ..

# 终端 2：启动 User API（端口 8081）
cd basebackend-user-api
mvn spring-boot:run -Dspring-boot.run.profiles=dev
cd ..

# 终端 3：启动 System API（端口 8082）
cd basebackend-system-api
mvn spring-boot:run -Dspring-boot.run.profiles=dev
cd ..
```

### 第五步：启动前端

```bash
cd basebackend-admin-web
npm install
npm run dev
```

## Docker Compose 一键启动

### 完整环境启动

```bash
# 1. 基础设施
cd docker/compose/base
docker-compose up -d

# 2. 中间件
cd ../middleware
docker-compose up -d

# 3. 可观测性（可选）
cd ../observability
docker-compose up -d

# 4. 微服务
cd ../services
docker-compose up -d
```

## 本地开发启动

### 最小启动集

本地开发只需启动以下核心服务：

| 服务 | 端口 | 说明 | 是否必须 |
|------|------|------|---------|
| MySQL | 3306 | 主数据库 | 必须 |
| Redis | 6379 | 缓存与分布式锁 | 必须 |
| Gateway | 8180 | API 网关 | 推荐 |
| User API | 8081 | 用户服务 | 必须 |
| System API | 8082 | 系统管理服务 | 必须 |

### 可选服务

| 服务 | 端口 | 说明 |
|------|------|------|
| Nacos | 8848 | 注册/配置中心（本地可用本地配置替代） |
| RocketMQ | 9876 | 消息队列（不启动时消息功能不可用） |
| Seata | 8091 | 分布式事务（已默认禁用） |
| MinIO | 9000/9001 | 文件存储 |
| Prometheus | 9090 | 监控 |
| Grafana | 3000 | 监控仪表盘 |
| Loki | 3100 | 日志聚合 |
| Tempo | 3200 | 链路追踪 |

## 常见启动问题排查

### 1. 端口冲突

```
Web server failed to start. Port XXXX was already in use.
```

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :8081
# 终止进程
kill -9 <PID>
```

### 2. MySQL 连接失败

```
Communications link failure
```

**检查清单**：
- 确认 MySQL 已启动：`docker ps | grep mysql`
- 确认数据库已创建：`mysql -u root -p -e "SHOW DATABASES;"`
- 确认连接参数正确（host / port / username / password）

### 3. Redis 连接失败

```
Unable to connect to Redis
```

**检查清单**：
- 确认 Redis 已启动：`redis-cli ping`
- 确认端口未被防火墙阻止
- 检查 Redis 密码配置

### 4. Flyway 迁移失败

```
Flyway migration failed
```

**解决方案**：
```bash
# 检查迁移脚本语法
cd docker/compose/flyway/sql
ls -la

# 如果需要修复已损坏的迁移记录
mysql -u root -p basebackend -e "DELETE FROM flyway_schema_history WHERE success=0;"
```

### 5. Maven 构建失败

```
Could not resolve dependencies
```

**解决方案**：
```bash
# 清理本地仓库并重新构建
mvn clean install -DskipTests -U

# 如果是网络问题，检查 Maven 镜像配置
cat ~/.m2/settings.xml
```

### 6. Lombok 注解无效

**解决方案**：
- 确认 IDE 已安装 Lombok 插件
- 确认已开启注解处理器（Annotation Processing）
- 重启 IDE

### 7. Seata 连接超时

```
can not connect to seata server
```

**说明**：Seata 默认配置为 `enabled: true`，如果未启动 Seata Server，将导致启动延迟。

**解决方案**：
- 如果不需要分布式事务：在 `application.yml` 中设置 `seata.enabled: false`
- 如果需要：启动 Seata Server `cd docker/seata-server && docker-compose up -d`
