# 监控告警

---

## 概述

BaseBackend 基于 Prometheus + Grafana + Loki 构建完整的监控告警体系，支持指标监控、日志告警和链路追踪告警。

---

## Prometheus 告警规则

### 告警规则文件

创建 `docker/compose/observability/alert-rules.yml`：

```yaml
groups:
  # ========================================
  # 服务可用性告警
  # ========================================
  - name: service_availability
    rules:
      - alert: ServiceDown
        expr: up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "服务 {{ $labels.instance }} 不可用"
          description: "{{ $labels.job }} 服务已宕机超过 1 分钟"

      - alert: HighErrorRate
        expr: |
          rate(http_server_requests_seconds_count{status=~"5.."}[5m])
          / rate(http_server_requests_seconds_count[5m]) > 0.05
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务 {{ $labels.application }} 错误率过高"
          description: "5xx 错误率超过 5%，当前值: {{ $value | humanizePercentage }}"

  # ========================================
  # 性能告警
  # ========================================
  - name: performance
    rules:
      - alert: HighResponseTime
        expr: |
          histogram_quantile(0.95,
            rate(http_server_requests_seconds_bucket[5m])
          ) > 2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务 {{ $labels.application }} 响应时间过高"
          description: "P95 响应时间超过 2 秒，当前值: {{ $value | humanizeDuration }}"

      - alert: HighRequestRate
        expr: rate(http_server_requests_seconds_count[1m]) > 1000
        for: 5m
        labels:
          severity: info
        annotations:
          summary: "服务 {{ $labels.application }} 请求量过高"
          description: "每秒请求量超过 1000，当前值: {{ $value }}"

  # ========================================
  # JVM 告警
  # ========================================
  - name: jvm
    rules:
      - alert: HighHeapUsage
        expr: |
          jvm_memory_used_bytes{area="heap"}
          / jvm_memory_max_bytes{area="heap"} > 0.85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "JVM 堆内存使用率过高"
          description: "{{ $labels.application }} 堆内存使用率超过 85%"

      - alert: HighGCPause
        expr: |
          rate(jvm_gc_pause_seconds_sum[5m])
          / rate(jvm_gc_pause_seconds_count[5m]) > 0.5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "GC 暂停时间过长"
          description: "{{ $labels.application }} 平均 GC 暂停时间超过 500ms"

      - alert: TooManyThreads
        expr: jvm_threads_live_threads > 500
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "活跃线程数过多"
          description: "{{ $labels.application }} 活跃线程数: {{ $value }}"

  # ========================================
  # 数据库告警
  # ========================================
  - name: database
    rules:
      - alert: HighDBConnectionUsage
        expr: |
          hikaricp_connections_active
          / hikaricp_connections_max > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "数据库连接池使用率过高"
          description: "{{ $labels.application }} 连接池使用率超过 80%"

      - alert: DBConnectionTimeout
        expr: rate(hikaricp_connections_timeout_total[5m]) > 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "数据库连接超时"
          description: "{{ $labels.application }} 出现数据库连接超时"

  # ========================================
  # Redis 告警
  # ========================================
  - name: redis
    rules:
      - alert: LowCacheHitRate
        expr: |
          rate(cache_gets_total{result="hit"}[5m])
          / rate(cache_gets_total[5m]) < 0.8
        for: 10m
        labels:
          severity: info
        annotations:
          summary: "缓存命中率过低"
          description: "{{ $labels.application }} 缓存命中率低于 80%"

  # ========================================
  # 磁盘告警
  # ========================================
  - name: disk
    rules:
      - alert: HighDiskUsage
        expr: |
          disk_used_bytes / disk_total_bytes > 0.85
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "磁盘使用率过高"
          description: "磁盘使用率超过 85%"
```

### Prometheus 配置

```yaml
# docker/compose/observability/prometheus.yml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

rule_files:
  - "alert-rules.yml"

alerting:
  alertmanagers:
    - static_configs:
        - targets:
            - alertmanager:9093

scrape_configs:
  - job_name: 'gateway'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8180']

  - job_name: 'user-api'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8081']

  - job_name: 'system-api'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8082']
```

## Grafana 看板配置

### 添加数据源

1. 登录 Grafana（http://localhost:3000，admin/admin）
2. 进入 `Configuration > Data Sources`
3. 添加以下数据源：

| 数据源 | 类型 | URL |
|--------|------|-----|
| Prometheus | Prometheus | `http://prometheus:9090` |
| Loki | Loki | `http://loki:3100` |
| Tempo | Tempo | `http://tempo:3200` |

### 推荐仪表盘

| 仪表盘 | Grafana Dashboard ID | 说明 |
|--------|---------------------|------|
| JVM (Micrometer) | 4701 | JVM 内存、GC、线程 |
| Spring Boot Statistics | 12900 | Spring Boot 应用指标 |
| MySQL Overview | 7362 | MySQL 监控 |
| Redis Dashboard | 763 | Redis 监控 |
| Node Exporter | 1860 | 主机监控 |

### 导入仪表盘

1. 进入 `Dashboards > Import`
2. 输入 Dashboard ID
3. 选择对应的数据源
4. 点击 Import

## 日志告警

### Grafana 日志告警规则

通过 Grafana Alerting 对 Loki 日志创建告警：

#### 错误日志告警

```
告警名称: Error Log Spike
数据源: Loki
查询: count_over_time({service=~"basebackend-.*"} |= "ERROR" [5m])
条件: > 10（5 分钟内超过 10 条 ERROR 日志）
```

#### OOM 告警

```
告警名称: OutOfMemoryError
数据源: Loki
查询: count_over_time({service=~"basebackend-.*"} |= "OutOfMemoryError" [1m])
条件: > 0
严重级别: Critical
```

#### 慢 SQL 告警

```
告警名称: Slow SQL Detected
数据源: Loki
查询: count_over_time({service=~"basebackend-.*"} |= "slow sql" [5m])
条件: > 5
```

## 告警通知渠道

### 配置 Alertmanager

```yaml
# alertmanager.yml
global:
  smtp_smarthost: 'smtp-mail.outlook.com:587'
  smtp_from: 'alert@basebackend.com'
  smtp_auth_username: '${ALERT_EMAIL_USERNAME}'
  smtp_auth_password: '${ALERT_EMAIL_PASSWORD}'

route:
  group_by: ['alertname', 'severity']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'email-notifications'

  routes:
    - match:
        severity: critical
      receiver: 'critical-notifications'
      repeat_interval: 1h

receivers:
  - name: 'email-notifications'
    email_configs:
      - to: 'ops-team@example.com'
        send_resolved: true

  - name: 'critical-notifications'
    email_configs:
      - to: 'ops-team@example.com'
        send_resolved: true
    # 可以添加更多通知方式：钉钉、企业微信、Slack 等
```

### Grafana 通知渠道

在 Grafana 中配置 Contact Points：

| 渠道 | 配置 | 适用告警 |
|------|------|---------|
| Email | SMTP 配置 | 所有告警 |
| Webhook | HTTP POST URL | 自定义集成 |
| Slack | Webhook URL | 团队通知 |
| DingTalk | 机器人 Webhook | 钉钉群通知 |

## 监控指标 SLO

### 建议的 SLO 目标

| 指标 | 目标 | 告警阈值 |
|------|------|---------|
| 可用性 | 99.9% | < 99.5% |
| P95 延迟 | < 500ms | > 2s |
| P99 延迟 | < 1s | > 5s |
| 错误率 | < 0.1% | > 1% |
| 缓存命中率 | > 90% | < 80% |
| DB 连接使用率 | < 70% | > 80% |
| JVM 堆使用率 | < 70% | > 85% |

## 常用排查命令

```bash
# 查看服务日志
docker logs -f --tail 100 basebackend-gateway

# 查看 GC 日志
docker exec basebackend-user-api cat /logs/gc.log

# 查看 JVM 堆内存
docker exec basebackend-user-api jmap -heap 1

# 查看线程栈
docker exec basebackend-user-api jstack 1 > thread-dump.txt

# 查看 MySQL 慢查询
docker exec -it basebackend-mysql mysql -u root -p \
  -e "SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 10;"

# 查看 Redis 内存使用
docker exec basebackend-redis redis-cli info memory
```
