# 缓存模块详解

---

## 概述

`basebackend-cache` 模块基于 Redis + Redisson 3.24.3 实现企业级缓存能力，支持多级缓存、分布式锁、布隆过滤器、热 Key 检测等高级特性。

---

## 多级缓存（L1 + L2）

### 架构

```
请求 → L1 本地缓存 (Caffeine)
          │
     命中? ──── 是 ──→ 返回数据
          │
          否
          │
          ▼
     L2 分布式缓存 (Redis)
          │
     命中? ──── 是 ──→ 回填 L1 → 返回数据
          │
          否
          │
          ▼
     数据库查询 ──→ 回填 L2 + L1 → 返回数据
```

### 特点

| 层级 | 技术 | 速度 | 容量 | 一致性 |
|------|------|------|------|--------|
| L1 | Caffeine | 纳秒级 | 小（JVM 内存） | 最终一致 |
| L2 | Redis | 毫秒级 | 大（独立服务） | 强一致 |

### 配置

```yaml
# application-cache.yml
cache:
  multi-level:
    enabled: true
    l1:
      max-size: 10000
      expire-after-write: 5m
    l2:
      default-ttl: 30m
      key-prefix: "basebackend:"
```

## 注解缓存

### `@Cacheable` — 查询缓存

```java
/**
 * 查询用户信息，优先从缓存获取
 * 缓存 Key: user:info:#{id}
 */
@Cacheable(value = "user:info", key = "#id")
public UserVO getUserById(Long id) {
    return userMapper.selectById(id);
}
```

### `@CachePut` — 更新缓存

```java
/**
 * 更新用户信息后刷新缓存
 */
@CachePut(value = "user:info", key = "#dto.id")
public UserVO updateUser(UserUpdateDTO dto) {
    userMapper.updateById(toEntity(dto));
    return getUserById(dto.getId());
}
```

### `@CacheEvict` — 清除缓存

```java
/**
 * 删除用户后清除缓存
 */
@CacheEvict(value = "user:info", key = "#id")
public void deleteUser(Long id) {
    userMapper.deleteById(id);
}

/**
 * 清除用户相关的所有缓存
 */
@CacheEvict(value = "user:info", allEntries = true)
public void clearAllUserCache() {
    // 清除所有用户缓存
}
```

## 分布式锁 `@DistributedLock`

> 详见 [Common 公共模块详解](Common公共模块详解) common-lock 部分

### 基本使用

```java
@DistributedLock(
    key = "'order:create:' + #dto.userId",
    waitTime = 5,
    leaseTime = 30,
    timeUnit = TimeUnit.SECONDS
)
public OrderVO createOrder(OrderCreateDTO dto) {
    // 同一用户同一时间只能创建一个订单
    return orderService.create(dto);
}
```

### 编程式使用

```java
@Autowired
private DistributedLockService lockService;

public void process() {
    String lockKey = "my-lock-key";
    boolean acquired = lockService.tryLock(lockKey, 5, 30, TimeUnit.SECONDS);
    if (acquired) {
        try {
            // 业务逻辑
        } finally {
            lockService.unlock(lockKey);
        }
    }
}
```

## 布隆过滤器防穿透

### 问题场景

缓存穿透：查询一个不存在的数据，每次都穿透缓存直接查库。

### 解决方案

使用布隆过滤器预判数据是否存在：

```java
@Autowired
private BloomFilterService bloomFilterService;

public UserVO getUser(Long id) {
    // 先检查布隆过滤器
    if (!bloomFilterService.mightContain("user:bloom", id.toString())) {
        return null;  // 数据肯定不存在，直接返回
    }
    // 布隆过滤器认为可能存在，查缓存或数据库
    return userMapper.selectById(id);
}
```

## 热 Key 检测与缓解

### 问题场景

某个 Key 被高频访问（热点 Key），可能导致单个 Redis 节点压力过大。

### 解决方案

- 自动检测高频访问的 Key
- 热 Key 自动提升到 L1 本地缓存
- 增加随机过期时间，避免缓存雪崩

## 缓存预热

应用启动时预加载热点数据到缓存：

```java
@Component
public class CacheWarmer implements ApplicationRunner {

    @Autowired
    private DictService dictService;

    @Override
    public void run(ApplicationArguments args) {
        // 启动时预热字典数据
        dictService.loadAllDictsToCache();
        log.info("字典缓存预热完成");
    }
}
```

## 分布式数据结构

Redisson 提供的分布式数据结构：

| 数据结构 | Java 接口 | 用途 |
|---------|----------|------|
| List | `RList<V>` | 分布式列表 |
| Map | `RMap<K,V>` | 分布式哈希表 |
| Queue | `RQueue<V>` | 分布式队列 |
| Set | `RSet<V>` | 分布式集合 |
| SortedSet | `RSortedSet<V>` | 分布式有序集合 |

### 使用示例

```java
@Autowired
private RedissonClient redissonClient;

// 分布式 Map
RMap<String, UserVO> userMap = redissonClient.getMap("user:map");
userMap.put("alice", aliceVO);

// 分布式队列
RQueue<TaskVO> taskQueue = redissonClient.getQueue("task:queue");
taskQueue.offer(newTask);
TaskVO task = taskQueue.poll();
```
