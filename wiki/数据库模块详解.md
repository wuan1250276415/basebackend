# 数据库模块详解

---

## 概述

`basebackend-database` 模块封装了数据库相关的基础设施能力，基于 MyBatis Plus 3.5.5 和 Flyway 9.22.3，提供动态数据源、多租户、审计日志、敏感数据加密等企业级功能。

---

## MyBatis Plus 配置

### 全局配置

```yaml
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true    # 下划线转驼峰
    cache-enabled: false                   # 关闭二级缓存
    call-setters-on-nulls: true           # null 值也调用 setter
    jdbc-type-for-null: 'null'            # null 值 JDBC 类型
  global-config:
    db-config:
      id-type: ASSIGN_ID                  # 雪花算法主键
      logic-delete-field: deleted          # 逻辑删除字段
      logic-delete-value: 1               # 已删除值
      logic-not-delete-value: 0           # 未删除值
  mapper-locations: classpath*:mapper/**/*.xml
```

### 分页插件

```java
@Configuration
public class MybatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        // 乐观锁插件
        interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
        return interceptor;
    }
}
```

### 自动填充

```java
@Component
public class MetaObjectHandler implements com.baomidou.mybatisplus.core.handlers.MetaObjectHandler {

    @Override
    public void insertFill(MetaObject metaObject) {
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "createBy", Long.class, UserContextHolder.getUserId());
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
        this.strictUpdateFill(metaObject, "updateBy", Long.class, UserContextHolder.getUserId());
    }
}
```

## Flyway 数据库迁移

### 配置

```yaml
spring:
  flyway:
    enabled: true
    baseline-on-migrate: true
    baseline-version: 0
    locations: classpath:db/migration
    table: flyway_schema_history
    encoding: UTF-8
    validate-on-migrate: true
    clean-disabled: true          # 禁用 clean 防止误删数据
    out-of-order: false
```

### 迁移脚本规范

- 路径：`src/main/resources/db/migration/`
- 命名规范：`V{major}.{minor}__description.sql`
- 示例：

```
db/migration/
├── V1.0__create_user_tables.sql
├── V1.1__add_user_email_column.sql
├── V1.2__create_role_tables.sql
├── V2.0__create_order_tables.sql
└── V2.1__add_order_status_index.sql
```

### 编写规范

```sql
-- V1.0__create_user_tables.sql

CREATE TABLE IF NOT EXISTS `sys_user` (
    `id`          BIGINT       NOT NULL COMMENT '用户ID',
    `username`    VARCHAR(50)  NOT NULL COMMENT '用户名',
    `password`    VARCHAR(100) NOT NULL COMMENT '密码',
    `email`       VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    `phone`       VARCHAR(20)  DEFAULT NULL COMMENT '手机号',
    `status`      TINYINT      NOT NULL DEFAULT 1 COMMENT '状态(1:正常 0:禁用)',
    `dept_id`     BIGINT       DEFAULT NULL COMMENT '部门ID',
    `create_by`   BIGINT       DEFAULT NULL COMMENT '创建者',
    `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_by`   BIGINT       DEFAULT NULL COMMENT '更新者',
    `update_time` DATETIME     DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted`     TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除(0:未删 1:已删)',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`),
    KEY `idx_dept_id` (`dept_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 动态数据源 `@DS`

### 使用场景
- 读写分离：主库写、从库读
- 多数据库：不同业务使用不同数据库

### 使用示例

```java
@Service
public class UserServiceImpl implements UserService {

    /**
     * 写操作 - 使用主库
     */
    @DS("master")
    @Override
    public void createUser(UserCreateDTO dto) {
        userMapper.insert(toEntity(dto));
    }

    /**
     * 读操作 - 使用从库
     */
    @DS("slave")
    @Override
    public UserVO getUser(Long id) {
        return userMapper.selectById(id);
    }
}
```

### 配置

```yaml
spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://master-host:3306/basebackend
          username: root
          password: ${MYSQL_PASSWORD}
        slave:
          url: jdbc:mysql://slave-host:3306/basebackend
          username: readonly
          password: ${MYSQL_SLAVE_PASSWORD}
```

## 多租户支持

### 机制说明

基于字段的租户隔离，通过 MyBatis 拦截器自动在 SQL 中追加租户条件。

```sql
-- 原始 SQL
SELECT * FROM sys_user WHERE status = 1;

-- 自动追加租户条件
SELECT * FROM sys_user WHERE status = 1 AND tenant_id = 1001;
```

### 配置

```yaml
database:
  enhanced:
    multi-tenancy:
      enabled: true
      tenant-column: tenant_id
      # 不需要租户隔离的表
      ignore-tables:
        - sys_tenant
        - sys_config
```

## 审计日志

### 功能说明

自动记录数据的增删改操作，包括操作人、操作时间、变更内容。

```yaml
database:
  enhanced:
    audit:
      enabled: true
      # 需要审计的表
      tables:
        - sys_user
        - sys_role
        - sys_menu
```

## 敏感数据加密 `@Sensitive`

### 使用示例

```java
@Data
@TableName("sys_user")
public class User extends BaseEntity {

    private String username;

    /**
     * 手机号 - 加密存储
     */
    @Sensitive(type = SensitiveType.PHONE)
    private String phone;

    /**
     * 身份证号 - 加密存储
     */
    @Sensitive(type = SensitiveType.ID_CARD)
    private String idCard;

    /**
     * 邮箱 - 加密存储
     */
    @Sensitive(type = SensitiveType.EMAIL)
    private String email;
}
```

### 工作原理
- 写入时：自动加密敏感字段后存入数据库
- 读取时：自动解密敏感字段返回明文
- API 响应时：按脱敏规则返回（如手机号显示 `138****8888`）

## 读写分离 `@MasterOnly`

```java
/**
 * 强制走主库查询（需要实时一致性的场景）
 */
@MasterOnly
public User getLatestUser(Long id) {
    return userMapper.selectById(id);
}
```

## 慢 SQL 监控

自动检测超过阈值的 SQL 并记录日志 / 告警。

```yaml
database:
  enhanced:
    slow-sql:
      enabled: true
      threshold: 1000    # 超过 1 秒视为慢 SQL
      log-level: WARN
```

---

## 故障转移

当主数据源不可用时，自动切换到备用数据源。

```yaml
database:
  enhanced:
    failover:
      enabled: true
      check-interval: 30s
      max-retries: 3
```
