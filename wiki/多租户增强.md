# 多租户增强

> `database-multitenant` — 租户解析策略、租户感知缓存、租户生命周期管理

## 概述

在原有数据库级多租户隔离基础上，新增租户解析策略链、租户感知缓存、租户生命周期管理等企业级增强能力。

## 新增组件

```
database-multitenant/ (13 → 21 文件)
├── 租户解析策略
│   ├── TenantResolver           — 策略接口
│   ├── HeaderTenantResolver     — 从请求头解析 (X-Tenant-Id)
│   ├── DomainTenantResolver     — 从域名解析 (tenant1.example.com)
│   ├── PathTenantResolver       — 从路径解析 (/api/tenant1/...)
│   └── CompositeTenantResolver  — 组合策略 (优先级链)
├── 租户感知缓存
│   └── TenantAwareCacheManager  — 自动追加 tenant: 前缀
├── 租户生命周期
│   └── TenantLifecycleManager   — 租户创建/禁用/删除/迁移
└── 配置增强
    └── MultiTenantProperties    — 解析策略/缓存/忽略表配置
```

## 快速使用

### 租户解析配置

```yaml
basebackend:
  database:
    multitenant:
      enabled: true
      resolve-strategy: header    # header / domain / path / composite
      header-name: X-Tenant-Id
      ignore-tables:              # 不做租户过滤的全局表
        - sys_config
        - sys_dict
```

### 组合解析策略

```java
// 自动按优先级尝试: Header → Domain → Path
@Bean
public CompositeTenantResolver compositeTenantResolver() {
    return new CompositeTenantResolver(List.of(
        new HeaderTenantResolver("X-Tenant-Id"),
        new DomainTenantResolver(),
        new PathTenantResolver("/api/{tenantId}/")
    ));
}
```

### 租户感知缓存

```java
// 缓存 Key 自动追加租户前缀
// 租户 A: "tenant:A:user:123"
// 租户 B: "tenant:B:user:123"
@Autowired
private TenantAwareCacheManager cacheManager;
```

### 租户生命周期

```java
@Autowired
private TenantLifecycleManager lifecycleManager;

lifecycleManager.createTenant("new-tenant", config);
lifecycleManager.disableTenant("old-tenant");
lifecycleManager.migrateTenant("tenant-1", "new-datasource");
```
