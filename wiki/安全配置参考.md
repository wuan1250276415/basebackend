## XSS 防护

### 机制说明

使用 OWASP HTML Sanitizer 对用户输入进行清理，防止跨站脚本攻击。

### 防护点

1. **Gateway 层**：`HeaderSanitizationFilter` 清理请求头中的危险内容
2. **应用层**：输入参数自动转义 HTML 特殊字符
3. **响应层**：`SecurityHeadersFilter` 添加 `X-XSS-Protection` 头

### 配置

```yaml
security:
  xss:
    enabled: true
    # 排除不需要 XSS 过滤的路径
    excludes:
      - /api/system/richtext/**
```

### 代码示例

```java
// OWASP HTML Sanitizer 使用
import org.owasp.html.PolicyFactory;
import org.owasp.html.Sanitizers;

PolicyFactory policy = Sanitizers.FORMATTING.and(Sanitizers.LINKS);
String safeHtml = policy.sanitize(userInput);
```

## CSRF 防护

### 机制说明

由于采用 JWT 无状态认证，不使用 Cookie 传递 Session，因此 CSRF 风险较低。但仍有以下防护措施：

- API 请求使用 `Authorization` Header 传递 Token（非 Cookie）
- Gateway 的 `SecurityHeadersFilter` 添加 `X-Content-Type-Options: nosniff`
- 前端 SPA 使用 `fetch` / `axios` 发起请求，不会被跨域表单触发

### Spring Security 配置

```java
@Configuration
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // 无状态 API，禁用 CSRF
            .csrf(csrf -> csrf.disable())
            // 其他配置...
            ;
        return http.build();
    }
}
```

## SQL 注入防护

### 防护层级

| 层级 | 措施 |
|------|------|
| ORM 层 | MyBatis Plus 参数化查询（`#{}` 占位符） |
| 应用层 | 输入参数校验（`@Valid` / `@Validated`） |
| Database 模块 | SQL 注入检测拦截器 |
| 连接池 | Druid `wall` 过滤器 |

### MyBatis 安全实践

```xml
<!-- 安全：使用 #{} 参数化查询 -->
<select id="selectUser" resultType="User">
    SELECT * FROM sys_user WHERE username = #{username}
</select>

<!-- 危险：避免使用 ${} 拼接 SQL -->
<!-- <select id="selectUser">
    SELECT * FROM sys_user WHERE username = '${username}'
</select> -->
```

### Druid Wall 过滤器

```yaml
spring:
  datasource:
    druid:
      filter:
        wall:
          enabled: true
          config:
            # 禁止多语句执行
            multi-statement-allow: false
            # 禁止删除表
            drop-table-allow: false
            # 禁止 TRUNCATE
            truncate-allow: false
```

## 请求签名验证

### 适用场景

对于支付、订单等敏感接口，启用请求签名验证防止篡改。

### 签名流程

```
客户端                                   服务端
  │                                       │
  │  1. 构造签名字符串                      │
  │     appId + timestamp + nonce          │
  │     + method + path                    │
  │     + canonical query                  │
  │     + SHA-256(body)                    │
  │                                       │
  │  2. 使用密钥 HMAC-SHA256 签名           │
  │                                       │
  │  3. 请求头携带签名                      │
  │     X-App-Id: mall-client             │
  │     X-Signature: xxx                   │
  │     X-Timestamp: 1708675200            │
  │     X-Nonce: abc123                    │
  │  ─────────────────────────────────>    │
  │                                       │
  │                    4. SignatureVerifyFilter 验证
  │                       ├── 检查时间戳有效期
  │                       ├── 规范化 query 参数顺序
  │                       ├── 计算请求体 SHA-256 摘要
  │                       ├── 重构签名字符串
  │                       ├── 计算签名并比对
  │                       └── 验证通过则放行
```

签名串固定为：

```text
appId|timestamp|nonce|method|path|canonicalQuery|bodyDigest
```

其中：

- `canonicalQuery` 会对 query 参数名和值按字典序规范化后再用 `&` 拼接，避免仅通过调换参数顺序绕过验签。
- `bodyDigest` 为请求体原文的 `SHA-256` 十六进制摘要；空请求体使用空字节数组的摘要值。

### 配置

```yaml
gateway:
  security:
    signature:
      enabled: ${GATEWAY_SIGNATURE_ENABLED:false}
      secret-key: ${GATEWAY_SIGNATURE_SECRET:your-secret-key-here}
      timestamp-validity: ${GATEWAY_SIGNATURE_VALIDITY:300000}  # 5 分钟
      paths:
        - /api/payment/**
        - /api/orders/**
      whitelist:
        - /api/payment/callback/**
```

## 安全响应头

`SecurityHeadersFilter` 为所有响应添加安全头：

| 响应头 | 值 | 作用 |
|--------|-----|------|
| `X-Content-Type-Options` | `nosniff` | 防止 MIME 类型嗅探 |
| `X-Frame-Options` | `DENY` | 防止点击劫持 |
| `X-XSS-Protection` | `1; mode=block` | 浏览器 XSS 过滤 |
| `Strict-Transport-Security` | `max-age=31536000` | 强制 HTTPS |
| `Content-Security-Policy` | `default-src 'self'` | 内容安全策略 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 引用来源策略 |

## 登录安全

### 登录限速

防止暴力破解密码：

```yaml
security:
  login:
    # 同一账号连续失败次数限制
    max-attempts: 5
    # 锁定时间（分钟）
    lock-duration: 30
    # 同一 IP 限速
    ip-rate-limit:
      count: 20
      period: 60  # 每分钟最多 20 次
```

### 密码安全

```java
// 密码加密存储（BCrypt）
String encodedPassword = passwordEncoder.encode(rawPassword);

// 密码验证
boolean matches = passwordEncoder.matches(rawPassword, encodedPassword);
```
