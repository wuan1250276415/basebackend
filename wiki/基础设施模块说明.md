# 基础设施模块说明

---

## 模块总览

| 模块 | 路径 | 核心职责 |
|------|------|---------|
| JWT | `basebackend-jwt/` | 令牌生成、验证、刷新、黑名单、密钥轮换 |
| Database | `basebackend-database/` | 数据源管理、Flyway 迁移、多租户、审计 |
| Cache | `basebackend-cache/` | 多级缓存、分布式锁、布隆过滤器 |
| Security | `basebackend-security/` | Spring Security、动态权限、XSS 防护 |
| Security Starter | `basebackend-security-starter/` | 安全自动配置 |
| Logging | `basebackend-logging/` | 结构化日志、审计日志、脱敏 |
| Observability | `basebackend-observability/` | OpenTelemetry、Metrics、Tracing |
| Messaging | `basebackend-messaging/` | RocketMQ、事务消息、幂等消费 |
| Transaction | `basebackend-transaction/` | Seata AT 模式封装 |
| Nacos | `basebackend-nacos/` | 配置管理、服务发现、灰度 |
| Feign API | `basebackend-feign-api/` | Feign 客户端接口定义 |
| Feature Toggle | `basebackend-feature-toggle/` | Unleash / Flagsmith |
| Web | `basebackend-web/` | Web 层公共配置 |

## JWT 模块 (`basebackend-jwt`)

> 详见 [JWT 认证体系](JWT认证体系)

### 核心功能
- **双 Token 机制**：Access Token（短期）+ Refresh Token（长期）
- **Token 黑名单**：支持主动吊销 Token
- **密钥轮换**：支持定期自动轮换签名密钥
- **多设备管理**：同一用户多端登录管理
- **事件审计**：登录、登出、Token 刷新等安全事件记录

### 依赖
```xml
<dependency>
    <groupId>com.basebackend</groupId>
    <artifactId>basebackend-jwt</artifactId>
</dependency>
```

## Database 模块 (`basebackend-database`)

> 详见 [数据库模块详解](数据库模块详解)

### 核心功能
- **MyBatis Plus 配置**：自动驼峰映射、逻辑删除、ASSIGN_ID 主键
- **Flyway 迁移**：自动化数据库版本管理
- **动态数据源 `@DS`**：多数据源切换
- **多租户支持**：基于字段的租户隔离
- **审计日志**：自动记录数据变更
- **敏感数据加密 `@Sensitive`**：字段级加密存储
- **读写分离 `@MasterOnly`**：主从库路由
- **慢 SQL 监控**：自动检测并告警慢查询
- **SQL 注入防护**：输入参数安全校验
- **故障转移**：数据源自动切换

### 配置示例
```yaml
# application-database.yml
database:
  enhanced:
    multi-tenancy:
      enabled: false
    sql-statistics:
      enabled: false
```

## Cache 模块 (`basebackend-cache`)

> 详见 [缓存模块详解](缓存模块详解)

### 核心功能
- **多级缓存**：L1 本地 Caffeine + L2 Redis
- **注解缓存**：`@Cacheable` / `@CachePut` / `@CacheEvict`
- **分布式锁**：`@DistributedLock` 注解
- **限流**：`@RateLimit` 注解
- **布隆过滤器**：防缓存穿透
- **热 Key 检测**：自动识别并缓解热点 Key
- **缓存预热**：启动时预加载热点数据
- **分布式数据结构**：List / Map / Queue / Set / SortedSet
- **多序列化器**：JSON / Kryo / Protobuf

### 依赖
```xml
<dependency>
    <groupId>com.basebackend</groupId>
    <artifactId>basebackend-cache</artifactId>
</dependency>
```

## Security 模块 (`basebackend-security`)

> 详见 [安全配置参考](安全配置参考)

### 核心功能
- **Spring Security 集成**：基于过滤器链的安全框架
- **权限注解**：`@RequiresPermission` / `@RequiresRole`
- **数据权限**：`@DataScope` 行级数据过滤
- **JwtAuthenticationFilter**：请求级 JWT 验证
- **动态权限加载**：从数据库动态加载权限配置
- **Token 黑名单检查**：登出后 Token 立即失效
- **安全事件发布**：登录失败、权限拒绝等事件
- **登录限速**：防暴力破解

### 依赖
```xml
<dependency>
    <groupId>com.basebackend</groupId>
    <artifactId>basebackend-security-starter</artifactId>
</dependency>
```

## Logging 模块 (`basebackend-logging`)

### 核心功能
- **结构化日志**：JSON 格式输出，便于日志聚合分析
- **审计日志**：记录用户操作审计轨迹
- **日志脱敏**：自动对敏感字段（手机号、身份证等）脱敏
- **链路追踪集成**：日志中自动注入 TraceId

## Observability 模块 (`basebackend-observability`)

> 详见 [可观测性](可观测性)

### 核心功能
- **OpenTelemetry 集成**：统一的 Traces / Metrics / Logs 收集
- **Micrometer 指标**：JVM、HTTP、自定义业务指标
- **SLO 监控**：服务级别目标定义与监控
- **告警规则**：基于指标的自动告警

### 配置示例
```yaml
# application-observability.yml
management:
  endpoints:
    web:
      exposure:
        include: '*'
  endpoint:
    health:
      show-details: always
```

## Messaging 模块 (`basebackend-messaging`)

> 详见 [消息队列](消息队列)

### 核心功能
- **RocketMQ 5.2.0 集成**：生产者/消费者封装
- **事务消息**：分布式事务最终一致性
- **幂等消费**：基于消息 ID 的去重
- **死信队列**：消费失败消息处理
- **Webhook**：HTTP 回调通知

### 配置示例
```yaml
rocketmq:
  name-server: localhost:9876
  producer:
    group: basebackend-producer-group
    send-message-timeout: 3000
    retry-times-when-send-failed: 2
```

## Transaction 模块 (`basebackend-transaction`)

### 核心功能
- **Seata AT 模式封装**：自动补偿型分布式事务
- **事务服务组管理**：`basebackend-seata-group`
- **Undo Log**：自动记录和回放
- **降级开关**：Seata Server 不可用时回退本地事务

### 配置要点
```yaml
seata:
  enabled: true
  tx-service-group: basebackend-seata-group
  data-source-proxy-mode: AT
  service:
    vgroup-mapping:
      basebackend-seata-group: default
    grouplist:
      default: localhost:8091
```

## Nacos 模块 (`basebackend-nacos`)

### 核心功能
- **配置管理**：集中管理所有微服务配置
- **服务发现**：自动注册与发现微服务实例
- **灰度发布**：基于 Nacos 元数据的灰度路由

### 配置
```yaml
# application-nacos.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        username: nacos
        password: nacos
      config:
        server-addr: localhost:8848
        file-extension: yml
```
