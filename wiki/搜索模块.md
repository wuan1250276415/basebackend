# 搜索模块

> `basebackend-search` — Elasticsearch/OpenSearch 全文搜索，DSL 查询构建器，中文分词支持

## 概述

基于 RestClient 实现的轻量级搜索模块，不依赖 ES Java Client 库，直接调用 ES REST API。支持 Elasticsearch 7.x/8.x 和 OpenSearch。

## 模块结构

```
basebackend-search/
├── client/
│   ├── SearchClient              — 搜索操作接口
│   └── RestClientSearchClient    — RestClient 实现
│       索引管理: createIndex / deleteIndex / existsIndex
│       文档操作: indexDocument / bulkIndex / getDocument / deleteDocument
│       搜索: search / count
│       Basic Auth / 索引前缀隔离
├── query/
│   └── SearchQuery               — DSL 查询构建器
│       条件: must / should / mustNot / filter
│       9种条件类型: match / matchPhrase / multiMatch / term / terms
│                   range / prefix / wildcard / exists
│       分页: page(pageNum, pageSize)
│       排序: sortBy / sortByScore
│       高亮: highlight
│       Source过滤: includes / excludes
├── model/
│   ├── SearchResult<T>           — 泛型搜索结果
│   │   ├── SearchHit<T>          — 单条命中 (id/score/source/highlights)
│   │   └── AggregationBucket     — 聚合桶
│   └── IndexDefinition           — 索引定义 Builder
│       10种字段类型: TEXT / KEYWORD / INTEGER / LONG / FLOAT / DOUBLE
│                    BOOLEAN / DATE / OBJECT / NESTED
│       分片/副本/分词器配置
├── annotation/
│   ├── @Searchable               — 标记可搜索实体
│   └── @SearchField              — 标记搜索字段
└── config/
    ├── SearchProperties          — 配置属性
    └── SearchAutoConfiguration   — 自动配置 (basebackend.search.enabled=true)
```

## 快速使用

### 1. 启用模块

```yaml
basebackend:
  search:
    enabled: true
    uris:
      - http://localhost:9200
    username: elastic
    password: ${ES_PASSWORD}
    index-prefix: prod_
    default-analyzer: ik_max_word
    default-search-analyzer: ik_smart
    connect-timeout: 5s
    read-timeout: 30s
```

### 2. 定义索引

```java
IndexDefinition def = IndexDefinition.builder("articles")
    .textField("title", "ik_max_word")
    .textField("content", "ik_max_word")
    .keywordField("status")
    .keywordField("author")
    .dateField("createTime")
    .integerField("viewCount")
    .booleanField("published")
    .shards(3)
    .replicas(1)
    .build();

searchClient.createIndex(def);
```

### 3. 索引文档

```java
// 单条
searchClient.indexDocument("articles", "1", articleMap);

// 批量
searchClient.bulkIndex("articles", documentList);
```

### 4. DSL 查询

```java
SearchQuery query = SearchQuery.builder("articles")
    .must(Condition.match("title", "Java虚拟线程"))
    .must(Condition.match("content", "并发编程"))
    .filter(Condition.term("status", "published"))
    .filter(Condition.range("createTime", "2024-01-01", null))
    .mustNot(Condition.term("deleted", true))
    .highlight("title", "content")
    .sortBy("createTime", SortOrder.DESC)
    .page(1, 20)
    .build();

SearchResult<Article> result = searchClient.search(query, Article.class);

for (var hit : result.hits()) {
    System.out.println(hit.id() + " → " + hit.source().getTitle());
    System.out.println("高亮: " + hit.highlights().get("title"));
}
```

### 5. 注解标记

```java
@Searchable(index = "articles", shards = 3)
public class Article {
    @SearchField(type = FieldType.TEXT, analyzer = "ik_max_word")
    private String title;

    @SearchField(type = FieldType.KEYWORD)
    private String status;

    @SearchField(type = FieldType.DATE)
    private LocalDateTime createTime;
}
```

## 设计决策

- **RestClient 实现**: 避免 ES Java Client 强依赖，通过 REST API 兼容 ES 7.x/8.x 和 OpenSearch
- **索引前缀**: 支持多环境隔离（dev_articles / prod_articles）
- **IK 分词默认**: `ik_max_word` 索引时最大颗粒度，`ik_smart` 搜索时智能分词
- **Record 不可变模型**: SearchResult / SearchHit / AggregationBucket / Condition 均为 Record
