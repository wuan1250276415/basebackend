# 工作流模块

> `basebackend-workflow` — 轻量级工作流引擎，审批流/条件分支/超时处理/任务管理

## 概述

纯 Java 实现的工作流引擎，不依赖 Flowable/Activiti 等重型框架。适合中小规模审批场景，内存实现可替换为数据库持久化。

## 模块结构

```
basebackend-workflow/
├── model/           # 流程模型
│   ├── ProcessDefinition    — 流程定义 Builder
│   ├── ProcessNode          — 流程节点 (5种类型)
│   ├── NodeType             — START / APPROVAL / CONDITION / NOTIFY / END
│   ├── ConditionBranch      — 条件分支 Record
│   ├── ProcessInstance      — 流程实例（状态机 + 变量 + 审批历史）
│   ├── ProcessStatus        — RUNNING / COMPLETED / REJECTED / CANCELLED / SUSPENDED
│   └── ApprovalRecord       — 审批记录 Record
│       6种动作: APPROVE / REJECT / TRANSFER / ADD_SIGN / WITHDRAW / TIMEOUT
├── engine/          # 流程引擎
│   ├── ProcessEngine        — 核心执行引擎
│   │   流程定义: deploy / getDefinition / getAllDefinitions
│   │   流程实例: startProcess / approve / reject / transfer / cancel
│   │   自动推进: START→跳过, CONDITION→条件计算, NOTIFY→通知, END→完成
│   │   任务查询: getTasksByRole / getMyProcesses
│   └── ConditionEvaluator   — 条件表达式求值器
│       支持: == != > >= < <= 六种操作符
│       类型: 数值比较 + 字符串比较
│       默认分支: default / * 通配符
└── config/
    └── WorkflowAutoConfiguration — 自动配置 (basebackend.workflow.enabled=true)
```

## 快速使用

### 1. 启用模块

```yaml
basebackend:
  workflow:
    enabled: true
```

### 2. 定义审批流程

```java
@Autowired
private ProcessEngine processEngine;

// 请假审批流程 — 条件分支
ProcessDefinition def = ProcessDefinition.builder("leave", "请假审批")
    .description("3天以内主管审批，超过3天需HR审批")
    .startNode("start", "提交申请")
    .conditionNode("days_check", "天数判断", List.of(
        ConditionBranch.of("days <= 3", "leader"),
        ConditionBranch.of("days > 3", "hr")
    ))
    .approvalNode("leader", "主管审批", "ROLE_LEADER")
    .approvalNode("hr", "HR审批", "ROLE_HR")
    .endNode("end", "审批完成")
    .transition("start", "days_check")
    .transition("leader", "end")
    .transition("hr", "end")
    .build();

processEngine.deploy(def);
```

### 3. 发起流程

```java
ProcessInstance inst = processEngine.startProcess("leave", "张三",
    Map.of("days", 5, "reason", "年假"));

// 自动根据条件路由到 HR 审批节点
assert inst.getCurrentNodeId().equals("hr");
```

### 4. 审批操作

```java
// 通过
processEngine.approve(inst.getInstanceId(), "HR王经理", "同意请假");

// 驳回
processEngine.reject(inst.getInstanceId(), "HR王经理", "事由不充分");

// 转交
processEngine.transfer(inst.getInstanceId(), "HR王经理", "HR李主管", "请李主管处理");

// 取消
processEngine.cancel(inst.getInstanceId(), "张三");
```

### 5. 查询待办

```java
// 按角色查询待办任务
List<ProcessInstance> tasks = processEngine.getTasksByRole("ROLE_HR");

// 查询我发起的流程
List<ProcessInstance> myProcesses = processEngine.getMyProcesses("张三");
```

### 6. 多步审批

```java
ProcessDefinition def = ProcessDefinition.builder("expense", "报销审批")
    .startNode("start", "提交报销")
    .approvalNode("leader", "主管审批", "ROLE_LEADER")
    .approvalNode("finance", "财务审批", "ROLE_FINANCE")
    .approvalNode("cfo", "CFO审批", "ROLE_CFO", 60) // 60分钟超时
    .endNode("end", "审批完成")
    .transition("start", "leader")
    .transition("leader", "finance")
    .transition("finance", "cfo")
    .transition("cfo", "end")
    .build();
```

## 流程变量

```java
// 启动时设置
processEngine.startProcess("leave", "张三", Map.of("days", 5));

// 运行时修改
instance.setVariable("urgent", true);

// 类型安全获取
Integer days = instance.getVariable("days", Integer.class);
```

## 条件表达式语法

```
variable operator value

支持的操作符: ==  !=  >  >=  <  <=
支持的值类型: 数字、字符串（用引号包裹）
默认分支: default 或 *

示例:
  amount <= 5000          → 数值比较
  department == "技术部"   → 字符串比较
  score >= 90.5           → 浮点数比较
  default                 → 其他条件都不满足时
```

## 设计决策

- **纯 Java 实现**: 不依赖 Flowable/Activiti，启动快、零外部依赖
- **内存存储**: 默认实现，业务方可替换为 MyBatis 持久化
- **条件表达式**: 简单实用的比较表达式，不引入 SpEL/MVEL 重依赖
- **所有 Bean 支持覆盖**: `@ConditionalOnMissingBean` 允许业务层替换 ProcessEngine
