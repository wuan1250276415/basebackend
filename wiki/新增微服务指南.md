# 新增微服务指南

---

## 概述

本文档指导如何从零创建一个新微服务并集成到 BaseBackend 架构中。

---

## 完整步骤

### 第一步：创建 Maven 模块

在项目根目录下创建新模块目录：

```bash
mkdir -p basebackend-xxx-api/src/main/java/com/basebackend/xxx
mkdir -p basebackend-xxx-api/src/main/resources
mkdir -p basebackend-xxx-api/src/main/resources/mapper
mkdir -p basebackend-xxx-api/src/main/resources/db/migration
```

### 第二步：编写 pom.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>com.basebackend</groupId>
        <artifactId>basebackend-parent</artifactId>
        <version>1.0.0-SNAPSHOT</version>
    </parent>

    <artifactId>basebackend-xxx-api</artifactId>
    <packaging>jar</packaging>
    <name>BaseBackend XXX API</name>
    <description>XXX 微服务</description>

    <dependencies>
        <!-- Spring Boot Web -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- 公共模块 -->
        <dependency>
            <groupId>com.basebackend</groupId>
            <artifactId>common-core</artifactId>
        </dependency>
        <dependency>
            <groupId>com.basebackend</groupId>
            <artifactId>common-context</artifactId>
        </dependency>

        <!-- 基础设施 -->
        <dependency>
            <groupId>com.basebackend</groupId>
            <artifactId>basebackend-database</artifactId>
        </dependency>
        <dependency>
            <groupId>com.basebackend</groupId>
            <artifactId>basebackend-cache</artifactId>
        </dependency>
        <dependency>
            <groupId>com.basebackend</groupId>
            <artifactId>basebackend-security-starter</artifactId>
        </dependency>

        <!-- API 文档 -->
        <dependency>
            <groupId>com.github.xiaoymin</groupId>
            <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
        </dependency>

        <!-- MySQL -->
        <dependency>
            <groupId>com.mysql</groupId>
            <artifactId>mysql-connector-j</artifactId>
            <scope>runtime</scope>
        </dependency>

        <!-- Lombok -->
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <optional>true</optional>
        </dependency>

        <!-- Test -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

### 第三步：注册到父 POM

在根 `pom.xml` 的 `<modules>` 中添加：

```xml
<modules>
    <!-- 已有模块... -->
    <module>basebackend-xxx-api</module>
</modules>
```

### 第四步：创建启动类

```java
package com.basebackend.xxx;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.client.discovery.EnableDiscoveryClient;

@SpringBootApplication(scanBasePackages = {
    "com.basebackend.xxx",
    "com.basebackend.common",
    "com.basebackend.database",
    "com.basebackend.cache",
    "com.basebackend.security"
})
@EnableDiscoveryClient
public class XxxApplication {

    public static void main(String[] args) {
        SpringApplication.run(XxxApplication.class, args);
    }
}
```

### 第五步：配置文件

创建 `src/main/resources/application.yml`：

```yaml
spring:
  application:
    name: basebackend-xxx-api
  profiles:
    active: ${SPRING_PROFILES_ACTIVE:dev}
  main:
    allow-bean-definition-overriding: true

server:
  port: 8083    # 选择一个未被占用的端口

# Knife4j 配置
knife4j:
  enable: true
  openapi:
    title: XXX 服务 API
    description: XXX 微服务接口文档
    version: 1.0.0
  setting:
    language: zh_cn
```

创建 `src/main/resources/application-dev.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/basebackend?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: ${MYSQL_PASSWORD:your-password}
    driver-class-name: com.mysql.cj.jdbc.Driver
  data:
    redis:
      host: localhost
      port: 6379

logging:
  level:
    com.basebackend.xxx: debug
```

### 第六步：创建分层代码

```
com.basebackend.xxx/
├── controller/
│   └── XxxController.java
├── service/
│   ├── XxxService.java          # 接口
│   └── impl/
│       └── XxxServiceImpl.java  # 实现
├── mapper/
│   └── XxxMapper.java
├── entity/
│   └── XxxEntity.java
├── dto/
│   ├── XxxCreateDTO.java
│   └── XxxUpdateDTO.java
├── vo/
│   └── XxxVO.java
└── config/
    └── XxxConfig.java
```

### 第七步：创建 Flyway 迁移脚本

```sql
-- src/main/resources/db/migration/V1.0__create_xxx_table.sql
CREATE TABLE IF NOT EXISTS `xxx` (
    `id`          BIGINT       NOT NULL COMMENT '主键ID',
    `name`        VARCHAR(100) NOT NULL COMMENT '名称',
    `status`      TINYINT      NOT NULL DEFAULT 1 COMMENT '状态',
    `create_by`   BIGINT       DEFAULT NULL COMMENT '创建者',
    `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_by`   BIGINT       DEFAULT NULL COMMENT '更新者',
    `update_time` DATETIME     DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,
    `deleted`     TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 第八步：Nacos 注册

如果使用 Nacos 服务发现，添加 `bootstrap.yml`：

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_SERVER_ADDR:localhost:8848}
        username: nacos
        password: nacos
      config:
        server-addr: ${NACOS_SERVER_ADDR:localhost:8848}
        file-extension: yml
```

### 第九步：Gateway 路由配置

在 `basebackend-gateway/src/main/resources/application-routes.yml` 中添加路由：

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 新增 XXX 服务路由
        - id: xxx-api
          uri: lb://basebackend-xxx-api
          predicates:
            - Path=/api/xxx/**
          filters:
            - StripPrefix=1
            - name: CircuitBreaker
              args:
                name: xxxServiceCircuitBreaker
                fallbackUri: forward:/fallback/xxx
```

### 第十步：构建与验证

```bash
# 构建整个项目
mvn clean install -DskipTests

# 启动新服务
cd basebackend-xxx-api
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 验证
curl http://localhost:8083/actuator/health
# 访问 API 文档
open http://localhost:8083/doc.html
```
